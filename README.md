# Followers CMS — панель управления продуктами

Веб-админка, которая заменяет управление цифровыми продуктами через Google Sheets.
Каждый продукт = отдельный лист исходной таблицы; внутри — список **вариантов онбординга**
с тарифами, инапной сеткой (паками) и апсейлами.

Это **фронт-прототип**: данные хранятся в `localStorage` браузера и засеяны из таблицы
`Followers_Products.xlsx` (файл `seed-data.js`). Бэкенда пока нет — но структура данных уже
готова, чтобы в любой момент завести API/БД и синхронизировать вместо таблицы.

## Два режима — две роли

В сайдбаре вверху переключатель режимов. Это разделение зон ответственности:

| Режим | Кто работает | Что делает |
|-------|--------------|-----------|
| **💲 Каталог** | key-менеджер | Ведёт прайс-лист продуктов «как в Google-таблице»: цены, setup fee, trial, billing/variant, статус (вкл/выкл). Это источник правды по продуктам. |
| **🧩 Онбординги** | продакт / QA | Собирает варианты онбордингов из продуктов каталога (тарифы / паки / апсейлы). |

**Каталог** (новый раздел) полностью повторяет структуру исходной таблицы: по листу-проекту
(UpFollow, Followerse, Wordbeat, Planly, GrowthPeak, Stage/Recurly, FollowBoost), внутри —
секции (Подписки / апсейлы / Инапная сетка / промо) с инлайн-редактируемой таблицей. Key-менеджер
заходит, добавляет/правит строки — ровно как в табличке, но в одном месте с онбордингами.

**Связка:** в редакторе варианта онбординга поля *Product code* и *Billing ID* подсказывают
значения из каталога того же проекта (автокомплит) — продакт собирает онбординг прямо из того,
что завёл key-менеджер.

## Запуск

```bash
cd followers-cms
python3 -m http.server 8000
# открыть http://localhost:8000
```

Либо просто открыть `index.html` в браузере.

## Доступ (демо)

```
логин:  admin
пароль: followers2026
```

Креды зашиты в `app.js` (`CREDS`) — это заглушка-гейт, а не настоящая авторизация.
При подключении бэкенда логин нужно перенести на сервер.

## Что умеет

- **4 продукта** в сайдбаре: UpFollow, Followerse, Wordbeat, Planly (с количеством вариантов).
- **Карточки вариантов** со статусом (Активен / На стопе), variant key, ссылкой и счётчиками
  тарифов/паков/апсейлов.
- **Поиск** по названию / variant key / ссылке и **фильтр** по статусу.
- **Редактор варианта** (боковая панель): статус, название, variant key, ссылка, флоу отмены,
  VAT, локализация + языки, логика начисления, и вложенные редактируемые списки:
  - **Тарифы подписки** — продукт / цена / billing ID (Recurly или Truegate);
  - **Инапная сетка (паки)** — название + product code;
  - **Апсейлы** — основной/скидочный варианты с product code.
- **Создание / удаление** вариантов.
- **Каталог продуктов** (режим key-менеджера): инлайн-редактируемые таблицы по секциям,
  поиск, фильтр по статусу (вкл/выкл), добавление/удаление строк, автосохранение.
- **Экспорт** онбордингов + каталога в один JSON и **сброс** активного режима к исходным данным.

## Структура данных

Онбординги (`seed-data.js`):

```
products[]
  ├─ id, name, billingLabel
  └─ variants[]
       ├─ status, name, link, variantKey, vat, localizationOn, languages[],
       │  accrualLogic, cancelFlow
       ├─ plans[]   { name, price, billingId }
       ├─ packs[]   { name, code }
       └─ upsells[] { title, main, mainCode, discount, discountCode }
```

Каталог (`catalog-data.js`, сгенерирован из исходной таблицы):

```
products[]                       // лист таблицы = проект
  ├─ id, name
  └─ sections[]                  // Подписки / апсейлы / Инапная сетка / промо
       ├─ title
       └─ rows[] { name, productId, price, setupFee, trial,
                   billing, segment, status (on|off), comments }
```

## Файлы

| Файл           | Назначение                                            |
|----------------|-------------------------------------------------------|
| `index.html`   | разметка: экран логина, оболочка, редактор-дровер      |
| `styles.css`   | тёмная тема админки                                    |
| `app.js`       | вся логика: auth, режимы, рендер, редактирование, persist |
| `seed-data.js` | онбординги, сгенерированные из `Followers_Products.xlsx` |
| `catalog-data.js` | каталог-прайслист, сгенерирован из листов исходной таблицы |
